# 구역별 센서 로그 이력 조회

카테고리: 환경/센서
설명: 구역별 센서 로그 이력을 조회합니다.
Method: GET
URL: /api/v1/zones/{id}/sensors/logs
담당자: 우하
사용자: ADMIN
우선순위: 2

### Request

| key | 설명 | value 타입 | 옵션 | Nullable | 예시 |
| --- | --- | --- | --- | --- | --- |
| startDate | 시작 날짜 | string | 필수 | X | “2026-04-01” |
| endDate | 끝 날짜 | string | 필수 | X | “2026-04-02” |

**Query parameter**

/api/v1/zones/1/sensors/logs?startDate=2026-04-01&endDate=2026-04-02

### Response

| key | 설명 | value 타입 | 옵션 | Nullable | 예시 |
| --- | --- | --- | --- | --- | --- |
| status | 상태값 | string |  | X | "success" |
| message | 메시지 | string |  | X |  "센서 로그 이력 조회가 완료되었습니다." |
| searchQuery | 관리자 요청 | JSON(object) |  | X | { "zoneId": 1, "period": "2026-04-01 ~ 2026-04-02" } (이렇게 하면 안되지 않나) |
| totalCount | 검색 결과 | integer |  | X | 150 |
| sensorDataList | 센서 데이터 리스트 | array |  | X |  [ // 로그 목록도 이 리스트 구조를 재사용하면 편합니다!
      { "id": 5001, "sensorType": "TEMPERATURE", "value": 24.5, "timestamp": "2026-04-01T14:00:00" },
      { "id": 5002, "sensorType": "TEMPERATURE", "value": 24.8, "timestamp": "2026-04-01T14:10:00" }
      ....
    ] |
| sensorType | 센서 타입 | string |  | X | "TEMPERATURE" |
| value | 센서 값 | integer |  | X | 24.5, 24.8 |
| unit | 단위 | string |  | X | "°C" (필요한가?) |
| timestamp | 측정 시간 | string |  | X | "2026-04-02T14:00:00", "2026-04-02T14:10:00" |

**Example**

```jsx
{
  "status": "success",
  "message": "센서 로그 이력 조회가 완료되었습니다.",
  "data": {
    "searchQuery": { "zoneId": 1, "period": "2026-04-01 ~ 2026-04-02" },
    "totalCount": 150,
    "attendanceList": [ // 로그 목록도 이 리스트 구조를 재사용하면 편합니다!
      { "id": 5001, "sensorType": "TEMPERATURE", "value": 24.5, "timestamp": "2026-04-01T14:00:00" },
      { "id": 5002, "sensorType": "TEMPERATURE", "value": 24.8, "timestamp": "2026-04-01T14:10:00" }
      ....
    ]
  }
}
```
