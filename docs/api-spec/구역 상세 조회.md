# 구역 상세 조회

카테고리: 구역
설명: 특정 구역의 상세 정보를 조회한다.
Method: GET
URL: /api/v1/zones/{id}
담당자: 박성종
사용자: ADMIN
우선순위: 4

### Request

X

**Header**

| key | 설명 | 예시 |
| --- | --- | --- |
| Authorization | Bearer Access Token | "Bearer eyJhbGci..." |

**Query parameter**

X

**Example (Request Body)**

X

### Response

| key | 설명 | value 타입 | 옵션 | Nullable | 예시 |
| --- | --- | --- | --- | --- | --- |
| code | 상태값 | String |  | X | "success" |
| message | 메시지 | String |  | X | "정상 조회되었습니다." |
| data.id | 구역 ID | Long |  | X | 2 |
| data.name | 구역명 | String |  | X | "1층 회의실 A" |
| data.zoneType | 구역 유형 | String |  | X | "AREA" |
| data.description | 구역 설명 | String |  | O | "소회의실" |
| data.parentId | 상위 구역 ID | Long |  | O | 1 |
| data.parentName | 상위 구역명 | String |  | O | "1층" |
| data.deviceCount | 설치 장치 수 | Integer |  | X | 2 |
| data.createdAt | 생성일시 | String |  | X | "2026-03-02T09:00:00" |
| data.updatedAt | 수정일시 | String |  | X | "2026-04-01T10:00:00" |

**Example**

```json
{
  "code": "success",
  "message": "정상 조회되었습니다.",
  "data": {
    "id": 2,
    "name": "1층 회의실 A",
    "zoneType": "AREA",
    "description": "소회의실",
    "parentId": 1,
    "parentName": "1층",
    "deviceCount": 2,
    "createdAt": "2026-03-02T09:00:00",
    "updatedAt": "2026-04-01T10:00:00"
  }
}
```

### Status

| status | response content |
| --- | --- |
| 200 | 구역 상세 조회 성공 |
| 401 | Access Token 누락 또는 유효하지 않음 |
| 403 | 관리자 권한 없음 |
| 404 | 존재하지 않는 구역 |
