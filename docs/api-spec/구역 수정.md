# 구역 수정

카테고리: 구역
설명: 구역 정보를 수정한다.
Method: PUT
URL: /api/v1/zones/{id}
담당자: 박성종
사용자: ADMIN
우선순위: 4

### Request

| key | 설명 | value 타입 | 옵션 | Nullable | 예시 |
| --- | --- | --- | --- | --- | --- |
| name | 구역명 | String |  | O | "1층 대회의실 A" |
| description | 구역 설명 | String |  | O | "대회의실로 변경" |
| parentId | 상위 구역 ID | Long |  | O | 1 |

**Header**

| key | 설명 | 예시 |
| --- | --- | --- |
| Authorization | Bearer Access Token | "Bearer eyJhbGci..." |

**Query parameter**

X

**Example (Request Body)**

```json
{
  "name": "1층 대회의실 A",
  "description": "대회의실로 변경",
  "parentId": 1
}
```

### Response

| key | 설명 | value 타입 | 옵션 | Nullable | 예시 |
| --- | --- | --- | --- | --- | --- |
| code | 상태값 | String |  | X | "success" |
| message | 메시지 | String |  | X | "구역 정보가 수정되었습니다." |
| data.id | 구역 ID | Long |  | X | 2 |
| data.name | 구역명 | String |  | X | "1층 대회의실 A" |
| data.zoneType | 구역 유형 | String |  | X | "AREA" |
| data.description | 구역 설명 | String |  | O | "대회의실로 변경" |
| data.parentId | 상위 구역 ID | Long |  | O | 1 |
| data.updatedAt | 수정일시 | String |  | X | "2026-04-01T10:00:00" |

**Example**

```json
{
  "code": "success",
  "message": "구역 정보가 수정되었습니다.",
  "data": {
    "id": 2,
    "name": "1층 대회의실 A",
    "zoneType": "AREA",
    "description": "대회의실로 변경",
    "parentId": 1,
    "updatedAt": "2026-04-01T10:00:00"
  }
}
```

### Status

| status | response content |
| --- | --- |
| 200 | 구역 수정 성공 |
| 400 | 요청 값 형식 오류 |
| 401 | Access Token 누락 또는 유효하지 않음 |
| 403 | 관리자 권한 없음 |
| 404 | 존재하지 않는 구역 또는 상위 구역 ID |
| 409 | 동일 상위 구역 내 중복 구역명 |
