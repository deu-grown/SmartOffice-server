# 구역별 최신 센서 데이터 조회

카테고리: 환경/센서
설명: 구역별 최신 센서 데이터를 조회합니다.
Method: GET
URL: /api/v1/sensors/latest
담당자: 우하
사용자: ADMIN
우선순위: 2

### Request

| key | 설명 | value 타입 | 옵션 | Nullable | 예시 |
| --- | --- | --- | --- | --- | --- |
| zoneId | 구역 id | integer | 필수 | X | 1 |

**Query parameter**

GET /api/v1/sensors/latest?zoneId=1

### Response

| key | 설명 | value 타입 | 옵션 | Nullable | 예시 |
| --- | --- | --- | --- | --- | --- |
| status | 상태값 | string |  | X | "success" |
| message | 메시지 | string |  | X |  "최신 센서 데이터 조회가 완료되었습니다." |
| searchQuery | 관리자 요청 | JSON(object) |  | X |  { "zoneId": 1 } |
| totalCount | 검색 결과 | integer |  | X | 2 |
| sensorDataList | 센서 데이터 리스트 | array |  | X | {
      { "sensorType": "TEMPERATURE", "value": 24.5, "unit": "°C", "timestamp": "2026-04-02T14:30:00" },
      { "sensorType": "HUMIDITY", "value": 45, "unit": "%", "timestamp": "2026-04-02T14:30:05" }  } |
| sensorType | 센서 타입 | string |  | X | "TEMPERATURE", "HUMIDITY" |
| value | 센서 값 | integer |  | X | 24.5, 45 |
| unit | 단위 | string |  | X | "°C", “%” |
| timestamp | 측정 시간 | string |  | X | "2026-04-02T14:30:00", "2026-04-02T14:30:05" |

**Example**

```jsx
{
  "status": "success",
  "message": "최신 센서 데이터 조회가 완료되었습니다.",
  "data": {
    "searchQuery": { "zoneId": 1 },
    "totalCount": 2,
    "sensorDataList": [  
      { "sensorType": "TEMPERATURE", "value": 24.5, "unit": "°C", "timestamp": "2026-04-02T14:30:00" },
      { "sensorType": "HUMIDITY", "value": 45, "unit": "%", "timestamp": "2026-04-02T14:30:05" }
    ]
  }
}
```
