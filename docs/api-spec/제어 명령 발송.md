# 제어 명령 발송

카테고리: 제어 명령
설명: 제어 명령을 발송합니다.
Method: POST
URL: /api/v1/controls
담당자: 우하
사용자: ADMIN
우선순위: 3

### Request

| key | 설명 | value 타입 | 옵션 | Nullable | 예시 |
| --- | --- | --- | --- | --- | --- |
| zoneId | 제어할 구역 고유 ID | Integer | 필수 | No | 1 |
| deviceId | 제어할 장치 식별 번호 | String | 필수 | No | "AC-001" |
| command | 제어 명령 (동작) | String | 필수 | No | "POWER_ON" |
| value | 제어 설정값 (온도 등) | String | 선택 | Yes | "24" |

**Header**

| key | 설명 | 예시 |
| --- | --- | --- |
| Authorization | 관리자 인증 토큰 (Bearer 타입) | Bearer eyJhbGciOiJIUzI1Ni... |
| Content-Type | 데이터 전송 형식 | application/json |

**Query parameter**

| key | 설명 | 타입 | Nullable | 예시 |
| --- | --- | --- | --- | --- |
|  |  |  |  |  |
|  |  |  |  |  |
|  |  |  |  |  |

**Example (Request Body)**

```json
// Request Body
{
  "zoneId": 1,
  "deviceId": "AC-001",
  "command": "POWER_ON",
  "value": "24"
}
```

### Response

| key | 설명 | value 타입 | 옵션 | Nullable | 예시 |
| --- | --- | --- | --- | --- | --- |
| status | 응답 성공 여부 | String | 필수 | No | "success" |
| message | 결과 상세 메시지 | String | 필수 | No | "제어 명령이 발송되었습니다." |
| data.controlId | 생성된 명령 고유 ID | Integer | 필수 | No | 1001 |

**Example**

```json
// Response Body
{
  "status": "success",
  "message": "제어 명령이 발송되었습니다.",
  "data": { "controlId": 1001 }
}
```

### Status

| status | response content | 설명 |
| --- | --- | --- |
| 200 | OK | 성공 |
| 400 | Bad Request | 파라미터 오류 |
| 401 | Unauthorized | 인증 실패 |
| 404 | Not Found | 데이터 없음 |
