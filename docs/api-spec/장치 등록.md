# 장치 등록

카테고리: 장치
설명: 새로운 장치를 등록한다. MQTT 토픽은 zoneId, deviceId 기반으로 서버에서 자동 생성한다.
Method: POST
URL: /api/v1/devices
담당자: 우하
사용자: ADMIN
우선순위: 1

### Request

| key | 설명 | value 타입 | 옵션 | Nullable | 예시 |
| --- | --- | --- | --- | --- | --- |
| zoneId | 설치 구역 ID | Long |  | X | 1 |
| name | 장치명 | String |  | X | "1층 입구 NFC 리더기" |
| deviceType | 장치 유형 | String |  | X | "NFC_READER" |
| serialNumber | 시리얼 번호 | String |  | O | "SN-NFC-001" |

**Header**

| key | 설명 | 예시 |
| --- | --- | --- |
| Authorization | Bearer Access Token | "Bearer eyJhbGci..." |

**Query parameter**

X

**Example (Request Body)**

```json
{
  "zoneId": 1,
  "name": "1층 입구 NFC 리더기",
  "deviceType": "NFC_READER",
  "serialNumber": "SN-NFC-001"
}
```

### Response

| key | 설명 | value 타입 | 옵션 | Nullable | 예시 |
| --- | --- | --- | --- | --- | --- |
| code | 상태값 | String |  | X | "success" |
| message | 메시지 | String |  | X | "장치가 등록되었습니다." |
| data.id | 생성된 장치 ID | Long |  | X | 1 |
| data.name | 장치명 | String |  | X | "1층 입구 NFC 리더기" |
| data.deviceType | 장치 유형 | String |  | X | "NFC_READER" |
| data.serialNumber | 시리얼 번호 | String |  | O | "SN-NFC-001" |
| data.mqttTopic | 자동 생성된 MQTT 토픽 | String |  | X | "smartoffice/zones/1/devices/1/nfc" |
| data.zoneId | 설치 구역 ID | Long |  | X | 1 |
| data.zoneName | 설치 구역명 | String |  | X | "1층" |
| data.status | 장치 상태 | String |  | X | "ACTIVE" |
| data.createdAt | 등록일시 | String |  | X | "2026-04-01T09:00:00" |

**Example**

```json
{
  "code": "success",
  "message": "장치가 등록되었습니다.",
  "data": {
    "id": 1,
    "name": "1층 입구 NFC 리더기",
    "deviceType": "NFC_READER",
    "serialNumber": "SN-NFC-001",
    "mqttTopic": "smartoffice/zones/1/devices/1/nfc",
    "zoneId": 1,
    "zoneName": "1층",
    "status": "ACTIVE",
    "createdAt": "2026-04-01T09:00:00"
  }
}
```

### Status

| status | response content |
| --- | --- |
| 201 | 장치 등록 성공 |
| 400 | 요청 값 누락 또는 형식 오류 |
| 401 | Access Token 누락 또는 유효하지 않음 |
| 403 | 관리자 권한 없음 |
| 404 | 존재하지 않는 구역 ID |
| 409 | 이미 등록된 시리얼 번호 |
