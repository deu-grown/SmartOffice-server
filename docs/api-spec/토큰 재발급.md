# 토큰 재발급

카테고리: 인증
설명: 리프레쉬 토큰으로 토큰을 재발급합니다.
Method: POST
URL: /api/v1/auth/refresh
담당자: 박성종
사용자: ALL
우선순위: 1

### Request

| key | 설명 | value 타입 | 옵션 | Nullable | 예시 |
| --- | --- | --- | --- | --- | --- |
| refreshToken | 발급받은 Refresh Token | String | X | X | "eyJhbGci..." |

**Header**

X

**Query parameter**

X

**Example (Request Body)**

```json
{
  "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

### Response

| key | 설명 | value 타입 | 옵션 | Nullable | 예시 |
| --- | --- | --- | --- | --- | --- |
| code | 상태값 | String |  | X | "success" |
| message | 메시지 | String |  | X | "토큰이 재발급되었습니다." |
| data.accessToken | 새로 발급된 Access Token | String |  | X | "eyJhbGci..." |
| data.tokenType | 토큰 타입 | String |  | X | "Bearer" |
| data.expiresIn | Access Token 만료 시간(초) | Integer |  | X | 1800 |

**Example**

```jsx
{
  "code": "success",
  "message": "토큰이 재발급되었습니다.",
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "tokenType": "Bearer",
    "expiresIn": 1800
  }
}
```

### Status

| status | response content |
| --- | --- |
| 200 | Access Token 재발급 성공 |
| 400 | refreshToken 누락 |
| 401 | Refresh Token 유효하지 않음 (변조 등) |
| 403 | Refresh Token 만료 → 재로그인 필요 |
