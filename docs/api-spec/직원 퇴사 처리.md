# 직원 퇴사 처리

카테고리: 직원/인사관리
설명: 직원을 퇴사 처리한다. 실제 DB 삭제가 아닌 status를 INACTIVE로 변경한다.
Method: DELETE
URL: /api/v1/users/{id}
담당자: 박성종
사용자: ADMIN
우선순위: 2

### Request

X

**Header**

| key | 설명 | 예시 |
| --- | --- | --- |
| Authorization | Bearer Access Token | "Bearer eyJhbGci..." |

**Query parameter**

X

**Example (Request Body)**

X

### Response

| key | 설명 | value 타입 | 옵션 | Nullable | 예시 |
| --- | --- | --- | --- | --- | --- |
| code | 상태값 | String |  | X | "success" |
| message | 메시지 | String |  | X | "퇴사 처리되었습니다." |

**Example**

```json
{
  "code": "success",
  "message": "퇴사 처리되었습니다."
}
```

### Status

| status | response content |
| --- | --- |
| 200 | 퇴사 처리 성공 |
| 401 | Access Token 누락 또는 유효하지 않음 |
| 403 | 관리자 권한 없음 |
| 404 | 존재하지 않는 직원 |
| 409 | 이미 퇴사 처리된 직원 |
