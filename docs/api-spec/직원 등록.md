# 직원 등록

카테고리: 직원/인사관리
설명: 새로운 직원을 등록한다. 초기 비밀번호는 서버에서 사번으로 자동 설정한다.
Method: POST
URL: /api/v1/users
담당자: 박성종
사용자: ADMIN
우선순위: 2

### Request

| key | 설명 | value 타입 | 옵션 | Nullable | 예시 |
| --- | --- | --- | --- | --- | --- |
| employeeNumber | 사번 | String |  | X | "EMP002" |
| name | 이름 | String |  | X | "김동우" |
| email | 이메일 (로그인 ID) | String |  | X | "kim@grown.com" |
| role | 권한 | String |  | X | "USER" |
| position | 직급 | String |  | X | "사원" |
| departmentId | 부서 ID | Long |  | O | 1 |
| phone | 연락처 | String |  | O | "010-9876-5432" |
| hiredAt | 입사일 | String |  | O | "2026-03-02" |

**Header**

| key | 설명 | 예시 |
| --- | --- | --- |
| Authorization | Bearer Access Token | "Bearer eyJhbGci..." |

**Query parameter**

X

**Example (Request Body)**

```json
{
  "employeeNumber": "EMP002",
  "name": "김동우",
  "email": "kim@grown.com",
  "role": "USER",
  "position": "사원",
  "departmentId": 1,
  "phone": "010-9876-5432",
  "hiredAt": "2026-03-02"
}
```

### Response

| key | 설명 | value 타입 | 옵션 | Nullable | 예시 |
| --- | --- | --- | --- | --- | --- |
| code | 상태값 | String |  | X | "success" |
| message | 메시지 | String |  | X | "직원이 등록되었습니다." |
| data.id | 생성된 직원 ID | Long |  | X | 2 |
| data.employeeNumber | 사번 | String |  | X | "EMP002" |
| data.name | 이름 | String |  | X | "김동우" |
| data.email | 이메일 | String |  | X | "kim@grown.com" |
| data.role | 권한 | String |  | X | "USER" |
| data.position | 직급 | String |  | X | "사원" |
| data.department | 부서명 | String |  | O | "개발팀" |
| data.status | 재직 상태 | String |  | X | "ACTIVE" |
| data.hiredAt | 입사일 | String |  | O | "2026-03-02" |
| data.createdAt | 등록일시 | String |  | X | "2026-03-02T09:00:00" |

**Example**

```json
{
  "code": "success",
  "message": "직원이 등록되었습니다.",
  "data": {
    "id": 2,
    "employeeNumber": "EMP002",
    "name": "김동우",
    "email": "kim@grown.com",
    "role": "USER",
    "position": "사원",
    "department": "개발팀",
    "status": "ACTIVE",
    "hiredAt": "2026-03-02",
    "createdAt": "2026-03-02T09:00:00"
  }
}
```

### Status

| status | response content |
| --- | --- |
| 201 | 직원 등록 성공 |
| 400 | 요청 값 누락 또는 형식 오류 |
| 401 | Access Token 누락 또는 유효하지 않음 |
| 403 | 관리자 권한 없음 |
| 404 | 존재하지 않는 부서 ID |
| 409 | 이미 사용 중인 이메일 또는 사번 |
