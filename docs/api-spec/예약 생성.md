# 예약 생성

카테고리: 예약
설명: 예약을 생성한다. 예약 구역 ID와 예약 시작 시각, 종료 시각, 사용 목적을 적는다.
Method: POST
URL: /api/v1/reservations
담당자: 박성종
사용자: ALL
우선순위: 9

### Request

| key | 설명 | value 타입 | 옵션 | Nullable | 예시 |
| --- | --- | --- | --- | --- | --- |
| zoneId | 예약하고자 하는 구역 고유 ID | Integer | 필수 | No | 1 |
| startTime | 예약 시작 시각 (ISO 8601) | String | 필수 | No | "2026-04-02T14:00:00" |
| endTime | 예약 종료 시각 (ISO 8601) | String | 필수 | No | "2026-04-02T16:00:00" |
| purpose | 사용 목적 | String | 선택 | Yes | "캡스톤 디자인 회의" |

**Header**

| key | 설명 | 예시 |
| --- | --- | --- |
| Authorization | 사용자/관리자 인증 토큰 (Bearer) | Bearer eyJhbGciOiJIUz... |
| Content-Type | 데이터 전송 형식 | application/json |

**Query parameter**

X

**Example (Request Body)**

```json
{
  "zoneId": 1,
  "startTime": "2026-04-02T14:00:00",
  "endTime": "2026-04-02T16:00:00",
  "purpose": "캡스톤 디자인 회의"
}
```

### Response

| key | 설명 | value 타입 | 옵션 | Nullable | 예시 |
| --- | --- | --- | --- | --- | --- |
| status | 응답 성공 여부 | String | 필수 | No | "success" |
| message | 결과 메시지 | String | 필수 | No | "예약이 성공적으로 생성되었습니다." |
| data.reservationId | 생성된 예약 고유 ID | Integer | 필수 | No | 503 |

**Example**

```json
{
  "status": "success",
  "message": "예약이 성공적으로 생성되었습니다.",
  "data": {
    "reservationId": 503
  }
}
```

### Status

| status | response content | 설명 |
| --- | --- | --- |
| 200 | OK | 요청 성공 (조회, 수정, 삭제) |
| 201 | Created | 예약 생성 성공 |
| 400 | Bad Request | 예약 시간 중복 또는 잘못된 형식 |
| 401 | Unauthorized | 인증 실패 (토큰 만료 등) |
| 403 | Forbidden | 권한 없음 (본인 예약이 아닌 경우 등) |
| 404 | Not Found | 존재하지 않는 예약 번호 |
