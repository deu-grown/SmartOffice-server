# 자산 수정

카테고리: 자산관리
설명: 자산 정보를 수정한다. 허용 구역 변경 시 이후 RFID 감지부터 새 구역 기준으로 이탈 여부를 판단한다.
Method: PUT
URL: /api/v1/assets/{id}
담당자: 박성종
사용자: ADMIN
우선순위: 8

### Request

| key | 설명 | value 타입 | 옵션 | Nullable | 예시 |
| --- | --- | --- | --- | --- | --- |
| name | 자산명 | String |  | O | "맥북 프로 14인치" |
| category | 자산 분류 | String |  | O | "ELECTRONICS" |
| rfidUid | RFID 태그 UID | String |  | O | "RF:A1:B2:C3" |
| allowedZoneId | 허용 구역 ID | Long |  | O | 3 |
| assignedUserId | 담당자 직원 ID | Long |  | O | 2 |
| description | 자산 설명 | String |  | O | "업데이트된 설명" |
| status | 자산 상태 | String |  | O | "INACTIVE" |
| purchasedAt | 구입일 | String |  | O | "2025-01-15" |

**Header**

| key | 설명 | 예시 |
| --- | --- | --- |
| Authorization | Bearer Access Token | "Bearer eyJhbGci..." |

**Query parameter**

X

**Example (Request Body)**

```json
{
  "name": "맥북 프로 14인치",
  "allowedZoneId": 3,
  "assignedUserId": 2,
  "status": "ACTIVE"
}
```

### Response

| key | 설명 | value 타입 | 옵션 | Nullable | 예시 |
| --- | --- | --- | --- | --- | --- |
| code | 상태값 | String |  | X | "success" |
| message | 메시지 | String |  | X | "자산 정보가 수정되었습니다." |
| data.id | 자산 ID | Long |  | X | 1 |
| data.assetNumber | 자산 번호 | String |  | X | "ASSET-001" |
| data.name | 자산명 | String |  | X | "맥북 프로 14인치" |
| data.allowedZoneId | 허용 구역 ID | Long |  | O | 3 |
| data.allowedZoneName | 허용 구역명 | String |  | O | "1층 서버실" |
| data.assignedUserId | 담당자 직원 ID | Long |  | O | 2 |
| data.assignedUserName | 담당자 이름 | String |  | O | "김동우" |
| data.status | 자산 상태 | String |  | X | "ACTIVE" |
| data.updatedAt | 수정일시 | String |  | X | "2026-04-01T10:00:00" |

**Example**

```json
{
  "code": "success",
  "message": "자산 정보가 수정되었습니다.",
  "data": {
    "id": 1,
    "assetNumber": "ASSET-001",
    "name": "맥북 프로 14인치",
    "allowedZoneId": 3,
    "allowedZoneName": "1층 서버실",
    "assignedUserId": 2,
    "assignedUserName": "김동우",
    "status": "ACTIVE",
    "updatedAt": "2026-04-01T10:00:00"
  }
}
```

### Status

| status | response content |
| --- | --- |
| 200 | 자산 수정 성공 |
| 400 | 요청 값 형식 오류 |
| 401 | Access Token 누락 또는 유효하지 않음 |
| 403 | 관리자 권한 없음 |
| 404 | 존재하지 않는 자산, 구역 ID 또는 직원 ID |
| 409 | 이미 등록된 RFID UID |
