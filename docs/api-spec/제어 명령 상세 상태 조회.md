# 제어 명령 상세/상태 조회

카테고리: 제어 명령
설명: 제어 명령을 상세하게 조회하거나 상태를 조회합니다.
Method: GET
URL: /api/v1/controls/{id}
담당자: 우하
사용자: ADMIN
우선순위: 3

### Request

X

**Header**

| key | 설명 | 예시 |
| --- | --- | --- |
| Authorization | 관리자 인증 토큰 (Bearer 타입) | Bearer eyJhbGciOiJIUzI1Ni... |
| Content-Type | 데이터 전송 형식 | application/json |

**Query parameter**

X

**Example (Request Body)**

X

### Response

| key | 설명 | value 타입 | 옵션 | Nullable | 예시 |
| --- | --- | --- | --- | --- | --- |
| status | 응답 성공 여부 | String | 필수 | No | "success" |
| data.controlId | 명령 고유 식별 번호 | Integer | 필수 | No | 1001 |
| data.status | 명령 처리 상태 (PENDING/COMPLETED/FAILED) | String | 필수 | No | "COMPLETED" |
| data.requestTime | 명령 요청 시각 | String | 필수 | No | "2026-04-02T15:00:00" |
| data.resultMessage | 장치 실행 결과 메시지 | String | 선택 | Yes | "정상 작동 확인" |

**Example**

```json
{
  "status": "success",
  "message": "상세 조회가 완료되었습니다.",
  "data": {
    "controlId": 1001,
    "status": "COMPLETED",
    "requestTime": "2026-04-02T15:00:00",
    "resultMessage": "장치가 정상적으로 작동되었습니다."
  }
}
```

### Status

| status | response content | 설명 |
| --- | --- | --- |
| 200 | OK | 성공 |
| 400 | Bad Request | 파라미터 오류 |
| 401 | Unauthorized | 인증 실패 |
| 404 | Not Found | 데이터 없음 |
